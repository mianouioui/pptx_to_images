# v1.1.0 发布记录

> 日期：2026-05-31
> 版本：v1.1.0 — 新增导出进度显示

## 概述

v1.1.0 新增了导出过程的进度反馈，解决此前「导出耗时较长时界面无任何输出、看起来像卡死」的问题。默认渲染引擎、命令行参数与输出命名均保持不变。

## 新增

- **导出进度显示**：每个外部步骤（LibreOffice 转 PDF、PDF 转 PNG、macOS PDFKit、PowerPoint、Keynote）在终端下会显示转圈动画与**实时秒数**，完成后显示 `✓` 与该步骤耗时，失败则显示 `✗`。
  - 即使某一步耗时较长（例如 LibreOffice 首次冷启动），也能看到秒数在走、确认程序在工作。
  - 在非交互场景（重定向 / 管道）下自动降级为静态的「开始 / 完成（耗时）」文本行，不会用动画字符污染日志。
- **批量计数**：批量导出时，每个文件前会显示 `[当前/总数]` 进度。

## 调整

- `run_command` 统一为所有外部命令套上进度指示，长任务不再「看起来卡死」。
- 超时错误信息会注明超时秒数。
- 「找不到 LibreOffice」报错现在附带安装方式（Homebrew / 官网链接）；README 新增「安装 LibreOffice」小节（中 / 英 / 法）。
- 新增 `PPTX2IMG_ENGINE` 环境变量用于设置默认引擎（如 `PPTX2IMG_ENGINE=keynote`），双击启动器 / 交互模式无需传 `--engine` 也能固定用 Keynote；命令行 `--engine` 仍可临时覆盖。

## 关于默认引擎

经实测，macOS 上 LibreOffice 渲染正常（示例文件约 5 秒完成），「卡死」主要源于缺少进度反馈而非引擎问题，因此 **默认引擎仍为 LibreOffice**（macOS：LibreOffice → Keynote 兜底）。如需改用 Keynote，可随时通过 `--engine keynote` 指定（注意 Keynote 按原生分辨率导出，`--width` 不生效）。

## 验证

- `python3 -m py_compile pptx_to_images.py` 通过
- 非交互场景：实际导出 3 页演示文稿，分阶段进度行与耗时显示正确
- 交互（伪终端）场景：单元测试确认 spinner 动画刷新、`\r` 原地覆盖、成功落 `✓` / 失败落 `✗`

## 兼容性

- 默认行为、命令行参数与输出命名（`page001.png` …）均不变，从 v1.0.x 升级无需调整使用方式。
