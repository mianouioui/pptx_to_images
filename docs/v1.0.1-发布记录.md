# v1.0.1 发布记录

> 日期：2026-05-31
> 版本：v1.0.1 — 维护性修复

## 概述

v1.0.1 是一次维护性更新，集中修复了 `--force` 复用输出目录、macOS PDFKit 兜底渲染、批量失败处理以及临时目录清理等问题，并让两个启动器的行为更一致。命令行参数、默认行为与输出命名均保持不变。

## 修复

- **`--force` 不再残留多余尾页**：复用已有输出目录时，导出前会先清掉上一次遗留的 `pageNNN.png`。此前若源演示文稿改短，旧的多余页会残留，污染后续 `images_to_pptx` 的合成结果。
- **macOS PDFKit 兜底与其他引擎一致**：改为先渲染到临时子目录，再统一按页码整理、应用 `--force`，渲染失败时不再在输出目录留下半成品图片。
- **批量转换不再因单个文件失败而整体中止**：失败的文件会被跳过，并在结尾统一列出，只要有失败即以非零码退出。
- **临时目录清理更稳健**：各引擎的 `.raw_*` 工作目录改用 `try/finally` 清理，渲染失败后不再残留隐藏目录。
- **多文件 + `-o` 打开路径修正**：完成后打开输出目录时使用展开并解析后的绝对路径。

## 调整

- macOS 启动器 `pptx_to_images.command` 取消与 Python 入口重复的横幅，交互引导只显示一次。
- Windows 启动器 `pptx_to_images.cmd` 支持一次拖入多个文件（之前仅处理第一个）。
- `--engine keynote` 在 `--help` 与运行时说明 `--width` 不生效（Keynote 按幻灯片原生分辨率导出）。
- `iter_presentations` 先按扩展名过滤再访问文件系统，减少多余的 `stat` 调用。

## 验证

- `python3 -m py_compile pptx_to_images.py` 通过
- 组件测试：`--force` 清理旧页、移动完成后 `.raw_*` 临时目录清理、`unique_dir` 自增、拖入路径解析（含转义/引号/空格）
- 端到端：在 macOS 上以 `LibreOffice + pdftoppm` 实际渲染一个 2 页演示文稿导出 `page001.png` / `page002.png`
- 回归：向输出目录注入多余的 `page003.png` 后用 `--force` 重跑，确认多余页被清除且无 `.raw_*` 残留

## 兼容性

- 命令行参数与默认行为保持不变，从 v1.0.0 升级无需调整使用方式。
- 输出文件命名（`page001.png`、`page002.png` …）不变，可继续直接搭配 `images_to_pptx`。
