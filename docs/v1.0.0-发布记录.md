# v1.0.0 发布记录

> 日期：2026-05-31
> 版本：v1.0.0 — 初始发布

## 概述

`pptx_to_images` 首次发布。该工具用于把 PowerPoint `.pptx` / `.ppt` 演示文稿逐页导出为 PNG 图片，输出文件按页码命名为 `page001.png`、`page002.png`，可直接搭配 `images_to_pptx` 使用。

## 新增功能

| 类型 | 支持内容 |
|------|----------|
| 输入文件 | `.pptx`、`.ppt` |
| 输出图片 | PNG |
| 批量处理 | 支持输入文件夹与 `--recursive` 递归扫描 |
| 输出命名 | `page001.png`、`page002.png`、`page003.png` |
| 渲染引擎 | PowerPoint、LibreOffice、macOS PDFKit、Keynote 兜底 |
| 运行方式 | Python 源码、macOS 启动器、Windows 启动器 |

### 核心特性

- 默认输出到 PPTX 文件旁边的 `<文件名>_images/` 文件夹
- 输出文件夹已存在时自动生成 `_2`、`_3` 后缀，避免误覆盖
- `--force` 可使用已有输出文件夹并覆盖同名图片
- `--width` 可指定图片宽度，默认 `1920px`
- `--prefix` 可指定输出文件名前缀，默认 `page`
- `--dry-run` 可预览将要转换的文件
- `--open` 可在完成后打开输出文件夹
- Python 源码不依赖第三方 Python 包

## 与参考项目搭配

本工具负责：

```text
PPTX -> page001.png / page002.png / ...
```

参考项目 `images_to_pptx` 负责：

```text
page001.png / page002.png / ... -> PPTX
```

## 验证

- `python3 -m py_compile pptx_to_images.py` 通过
- 使用测试 PPTX 端到端导出 2 张 PNG
- macOS 上验证 `LibreOffice + pdftoppm` 路线
- macOS 上验证无 `pdftoppm` 时的 `LibreOffice + PDFKit` 兜底路线
- 使用 `images_to_pptx.py --dry-run` 验证导出的 `page001.png` / `page002.png` 可被参考项目识别

## 已知说明

- 逐页导出 PNG 需要真实渲染引擎；仅解析 PPTX XML 无法得到最终视觉效果。
- Windows PowerPoint 启动器依赖本机已安装 Microsoft PowerPoint，建议在真实 Windows 机器上再做发布前验证。
